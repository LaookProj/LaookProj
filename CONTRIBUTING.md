# Contributing to Laook

First off, thank you for considering contributing to Laook! It's people like you that make Laook such a great platform.

## 1. Getting Started
- Fork and clone the repositories you wish to contribute to (e.g., `laook-backend-api`, `laook-models`, `laook-mobile-app`, `laook-flutter-app`).
- Ensure you have the required dependencies installed (refer to each repository's `README.md`).

## 2. Branching Strategy
We use a standard Git Flow branching model.
- **`main`**: The primary branch. This branch is protected and requires a pull request to merge changes.
- **Feature Branches**: Create a new branch for each feature or bug fix. Use a descriptive naming convention: `feature/your-feature-name` or `bugfix/issue-description`.

## 3. Making Changes
- Write clear, concise, and well-documented code.
- Ensure your changes do not break existing functionality.
- Write tests for your changes if applicable.

## 4. Submitting a Pull Request (PR)
- Push your feature branch to your fork.
- Open a Pull Request against the `main` branch of the upstream repository.
- Ensure all CI/CD status checks (e.g., `ci/build`, `ci/lint`) pass.
- Request a review from the `developers` team. 
- At least **2 approvals** are required before a PR can be merged (unless exceptions apply for specific maintainers).

## 5. Code Review
- Be open to feedback and suggestions from reviewers.
- Make necessary updates to your PR based on the review.

## 6. Reporting Issues
If you find a bug or have a feature request, please open an issue in the relevant repository. Provide as much detail as possible, including steps to reproduce bugs.

Thank you for contributing!
